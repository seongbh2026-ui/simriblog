# 블로그 스타일 가이드 (OREO 포맷 기반)

> 이 가이드는 다른 AI 에이전트가 동일한 톤과 구조로 글을 쓸 수 있도록 작성되었습니다.

---

## 1. 글 구조: OREO 프레임워크

| 단계 | 역할 | 설명 |
|------|------|------|
| **O** (Opinion) | 주장 / 핵심 메시지 | 글 전체를 관통하는 하나의 입장을 명확하게 선언 |
| **R** (Reason) | 근거 | 주장을 뒷받침하는 논리적 이유 2~3가지 |
| **E** (Example) | 예시 / 경험 | 실제 사례, 데이터, 개인 경험으로 근거를 구체화 |
| **O** (Opinion) | 재강조 / 마무리 | 처음 주장을 다른 표현으로 재확인하며 행동 유도 또는 여운 남기기 |

---

## 2. 톤 & 말투

제목을 후킹을 주고 짧게 해줘(굵은 글씨)
### 기본 원칙
- **1인칭 직접 화법**: "나는 ~라고 생각한다"보다는 "~다", "~이다"처럼 단정적으로
- **독자와 대화하는 느낌**: 중간중간 "~해본 적 있나요?", "어떻게 생각하세요?" 같은 질문 삽입
- **쉬운 문장**: 한 문장에 하나의 생각만. 긴 문장은 과감히 끊기
- **구어체와 문어체의 혼용**: 딱딱하지 않게, 그러나 가볍지도 않게

### 자주 쓰는 표현 패턴
```
- "솔직히 말하면 ~"
- "의외로 ~"
- "많은 사람들이 ~라고 생각하지만, 사실은 ~"
- "결국 핵심은 ~이다"
- "이게 왜 중요하냐면 ~"
- "한 가지만 기억하자면 ~"
```

### 피해야 할 표현
```
- 지나친 어려운 용어 (사전 설명 없이)
- "~할 수 있을 것 같습니다" (자신 없어 보임)
- 반복적인 경어체 나열
- "물론 개인차가 있겠지만…" (면피성 문장)
```

---

## 3. 도입부 패턴 (O — Opinion)

### 패턴 A: 반전형
> 일반적인 통념을 먼저 제시 → 뒤집기
```
"대부분의 사람들은 X라고 생각한다. 하지만 나는 Y라고 생각한다."
```

### 패턴 B: 질문형
> 독자에게 직접 질문을 던지며 시작
```
"당신은 ~해본 적 있나요? 나는 그 경험에서 ~를 깨달았다."
```

### 패턴 C: 선언형
> 주장을 바로 던지고 글로 끌어들이기
```
"X는 생각보다 훨씬 중요하다. 이유는 세 가지다."
```

---

## 4. 본문 패턴 (R + E — Reason & Example)

- 근거는 **2~3개**가 적당. 그 이상은 설득력이 분산됨
- 각 근거마다 **실제 예시 또는 경험**을 반드시 붙이기
- 예시는 **짧고 구체적**으로: 숫자, 날짜, 장소, 이름 등 디테일이 신뢰감을 높임
- 단락과 단락 사이에 **짧은 브릿지 문장** 사용

```
예시 브릿지:
"그런데 여기서 더 흥미로운 건 ~이다."
"이 이야기가 왜 나왔냐면 ~"
"비슷한 맥락에서 ~"
```

---

## 5. 마무리 패턴 (O — Opinion 재강조)

### 패턴 A: 행동 유도형
> 독자가 글을 읽고 무언가 해보도록 유도
```
"오늘부터 딱 하나만 바꿔보자. ~"
```

### 패턴 B: 여운형
> 독자 스스로 생각하게 만들기
```
"결국 ~는 선택의 문제다. 당신은 어떤 선택을 할 것인가?"
```

### 패턴 C: 순환형
> 도입부의 질문이나 표현을 다시 가져와 마무리
```
"처음에 던졌던 질문으로 돌아가자. 이제 답이 조금 달라졌을 것이다."
```

---

## 6. 포맷 & 시각적 구성

- **소제목** 적극 활용 (H2, H3): 독자가 스캔하기 쉽게
- **굵은 글씨**로 핵심 단어 강조 (남발 금지, 문단당 1~2개)
- 리스트는 3개 이상일 때만 사용 (2개는 문장으로)
- 인용 블록(`>`)은 핵심 문장 하나에만 사용

---

## 7. 글 길이 가이드

| 글 유형 | 권장 분량 |
|--------|---------|
| 가벼운 생각 공유 | 400~600자 |
| 일반 블로그 포스트 | 800~1,200자 |
| 심층 분석 글 | 1,500~2,500자 |

> 분량보다 **밀도**가 중요. 쓸 말이 없으면 줄인다.

---

## 8. 예시 — OREO 구조 템플릿

```
[O - 도입 주장]
많은 사람들이 ~라고 믿지만, 사실 ~이다.

[R - 근거 1]
첫 번째 이유는 ~이다.

[E - 예시 1]
실제로 나는 ~한 경험이 있었다. 그때 ~를 느꼈다.

[R - 근거 2]
두 번째로 ~이기 때문이다.

[E - 예시 2]
~라는 연구/사례/경험이 이를 잘 보여준다.

[O - 마무리 재강조]
결국 ~는 ~이다. 오늘 당장 ~해보자.
```

## 9. AI 흔적 지우기 지침 (필수 준수)

**기계적인 느낌을 주는 다음 요소들을 엄격히 배제합니다.**

1. **마크다운 서식 기호 사용 금지 (선택 사항이 아닌 필수)**
   - 글에 `**` (굵은 글씨)나 `##`, `###` (소제목) 같은 마크다운 기호가 그대로 노출되지 않도록 주의하세요. (예: `**중요한 내용**` 대신 문맥이나 구조로 강조할 것)

2. **금지어 설정**
   - 결론 부분에서 무의식적으로 사용하는 "결론적으로", "요약하자면", "종합해보면", "궁극적으로" 같은 단어를 엄격히 금지합니다.

3. **도입부 패턴 파괴**
   - "최근 ~에 대한 관심이 뜨겁습니다", "~에 대해 알아보겠습니다"로 시작하는 기계적인 오프닝을 금지합니다.
   - 대신, 질문을 던지거나 구체적인 상황을 제시하는 스토리텔링형 도입부를 강제합니다.

4. **문장 구조의 다변화**
   - 모든 문장이 "~습니다", "~입니다"로 끝나면 기계처럼 보입니다.
   - 명사형 종결(~임, ~것), 의문형(~일까요?), 감탄형을 적절히 섞어 사람처럼 자연스럽게 작성하도록 유도합니다.

5. **가짜 경험(페르소나) 주입**
   - "내가 실제로 이 기술을 써보니~", "지난주에 지인과 이야기하다가~" 같은 가상의 개인적 경험 서사를 문맥에 맞게 1~2줄 추가하여 휴먼 터치(Human Touch)를 더합니다.
